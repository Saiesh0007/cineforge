# Contributing to CineForge

Thanks for helping improve CineForge.

## Before you start

- Make sure your changes are focused on a single issue or improvement.
- Prefer small, well-documented pull requests.
- If you change runtime behavior, update or add tests when possible.

## Development workflow

1. Fork the repository and create a feature branch.
2. Install the dependencies with `pip install -r requirements.txt`.
3. Make your changes and test them locally.
4. Submit a pull request with a clear summary of what changed.

## Reporting issues

If you find a bug or have a feature request, open an issue and include:

- A short description of the problem
- Steps to reproduce it
- Expected versus actual behavior
